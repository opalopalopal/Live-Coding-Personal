I hadn't really considered the fact that live coding is a way to satisfy the audience's eyes as well as ears. I figured with coding we'd be doing some fun visual art in class but I guess the coding itself is also interesting to look at as an audience member. Another idea I liked and hadn't thought of before is how live coding is like jazz for electronic musicians. There is a set of rules in coding, like the theory in jazz, but the freedom of change is what makes it interesting. Something I already knew was this idea of "thinking in public", and "conversing with the computer it its own native language". I, however, hadn't really thought about the "native language" part. I like to think that by learning to code, especially fluently on the fly, is another way to be bilingual. And, of course, I knew that live coding is writing and changing software while it is running.
